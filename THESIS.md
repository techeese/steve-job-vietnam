> **What this file is.** The *immutable ideal* the game is measured against — the question it
> exists to answer and the laws a real answer must obey. It is deliberately split out from the
> *implementation* (`DESIGN.md`, `ROADMAP.md`, the craft half of `VISION.md`), which is disposable
> and re-derivable. The autonomous loop measures the **gap** between the live game and THIS file.
>
> **Authority.** This file is **owner-owned**. The autonomous loop MUST NOT edit §A–§E. It may only
> *propose* a frame change as an `OWNER:` veto line and wait for the owner to apply it. Everything

## §A — The question (đề Văn 2026, verbatim)

> **"Làm thế nào để Việt Nam có những Steve Jobs Việt Nam?"**
> *(How does Vietnam grow its own Steve Jobs?)*

The game is a **playable essay** on this question. It does not answer it. It lets the player
**arrive at their own answer** by founding a school, setting its philosophy, and watching the
little people grow up and scatter into the world.

## §B — The open-question law (the game must stay an *essay*, not a *lecture*)

1. **No single dominant strategy.** Every viable path is a distinct, defensible *thesis* about how
   talent is made or wasted. If one strategy strictly wins, the essay has collapsed into an answer.
2. **Reflect consequences; never impose a verdict.** The game shows what each choice does to real
   lives. It does not tell the player which choice was right.
3. **The ending mirrors the player back.** The closing essay is assembled from *their* graduates —
   the question answered by the faces that crossed their sân, never by a stated moral. That essay is the
   game's true payoff: it should be **long, personal to the run, and genuinely moving** — the headmaster
   thinking aloud, built across annual letters to the climactic *"how to have a Steve Jobs Vietnam"* essay
   when the first 🍎 appears. **It wrestles with the question; it never answers it.**

## §C — The four person-sim invariants (design law — these protect the charter)

1. **Whose-life, not which-strategy.** Talent changes *whose* life each thesis realizes or wastes —
   never *which thesis is correct*.
2. **Symmetry of waste.** Every preset must REALIZE some talents and WASTE others. No preset is
   waste-only or realize-only; waste is never illustrated *exclusively* by cram.
3. **Prose, not a meter.** The realize/waste read surfaces as **caused prose tied to a moment** —
   never a persistent number, bar, or sortable 48-kid efficiency table. *A number invites a
   spreadsheet; a sentence invites grief.*
4. **Waste is done TO the person, never their deficiency.** Wasted/distorted prose blames the
   SYSTEM and the player's choices, never the kid.

## §D — The marks of a true answer (what a real answer *feels* like)

A real answer to §A is not a feature list — it is an **experience**. These are the few marks that
tell you the game has *become* that experience. They are written as the **ideal**, free of any
mechanic (no rooms, no stats, no specific outcomes — those live in the implementation and change).
This file says only what "true" is; how the gap to it is measured is the engine's job, not this file's.

**The test — have a stranger play a few years, then read five lives from the school they ran.**
If they would…

1. **…see a person become someone.** Each life shows a talent meeting your school and *turning into*
   a life — fulfilled, wasted, or twisted — and it's clear the **school** did it, not the child.
2. **…care.** At least one life *moves* them — they would grieve a wasted gift or cheer a fulfilled
   one, **by name** — because they feel the player's choices caused it.
3. **…find no single right way.** Different philosophies each produce a real, dignified kind of
   success — the quiet steady life as worthy as the **rare maker the question names**, who stays
   rare and hard-won, never handed out — and the game crowns none.
4. **…leave with their *own* answer.** The ending hands the question back: the school reflected, not
   a moral delivered. The satire bites — in a school **Vietnam would recognize** — and never lectures.
5. **…have felt the weight while playing.** Each philosophy was a real fork with uncertain stakes —
   *chosen*, not optimized — so the argument is **lived in the playing**, not only read in the epilogue.

…then the game is answering §A. If they would shrug, or feel *told what to think*, or find one
obvious "best" way to play — it is not, **no matter how good it looks.**

## §E — What this file is NOT

It is **not** the implementation. The numbers, the FSM, the art, the room list, the specific
mechanics — all live in `DESIGN.md` / `ROADMAP.md` / `engine.js` and are **disposable**. If the
implementation is beautiful but a mark in §D is missing, the implementation is wrong, not this
file. *Impressive ≠ true.* This file is "true"; the Bar rubric is "impressive."
